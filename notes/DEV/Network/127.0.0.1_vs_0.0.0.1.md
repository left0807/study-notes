## **127.0.0.1 - Localhost / Loopback Address**
```
My Web is listening on IPv4: 127.0.0.1:8000
```

- **"This computer only"** - Only accessible from the machine it's running on
- **Loopback interface** - A virtual network interface that loops back to itself
- **Cannot be accessed from:** Other computers, VPN connections, other devices on your network
- **Use case:** Development, security (don't expose to network), testing

## **0.0.0.0 - "All Interfaces" / Wildcard Address**
```
My Web is listening on IPv4: 0.0.0.0:8000
```

- **"Listen on ALL network interfaces"** - Makes the service available on every network interface the computer has
- **Includes:** Ethernet, WiFi, VPN interfaces, loopback, virtual interfaces
- **Can be accessed from:** Localhost, other computers on LAN, VPN connections, internet (if port is open)
- **Use case:** Production, remote access, serving to multiple clients

## **Practical Examples:**

### **When binding to 127.0.0.1:**
```bash
# Only these work:
curl http://127.0.0.1:8000/
curl http://localhost:8000/

# These DO NOT work:
curl http://192.168.1.100:8000/  # Your server's LAN IP
curl http://10.8.0.2:8000/       # Your VPN IP
curl http://YOUR_PUBLIC_IP:8000/
```

### **When binding to 0.0.0.0:**
```bash
# ALL of these work:
curl http://127.0.0.1:8000/
curl http://localhost:8000/
curl http://192.168.1.100:8000/  # Your server's LAN IP
curl http://10.8.0.2:8000/       # Your VPN IP
curl http://YOUR_PUBLIC_IP:8000/ # If port forwarded
```

## **Security Implications:**

**127.0.0.1 (Safer):**
- Only local users/apps can access
- Protected from network attacks
- Good for databases, admin interfaces

**0.0.0.0 (More exposed):**
- Accessible to anyone on network/internet
- Requires firewall rules for protection
- Good for web servers, APIs


## **How to check what's binding where:**
```bash
# See what interfaces programs are listening on
sudo ss -tlnp | grep :8000

# Output examples:
LISTEN 0 128    127.0.0.1:8000    0.0.0.0:*   # BAD for VPN access
LISTEN 0 128       0.0.0.0:8000    0.0.0.0:*   # GOOD for VPN access
LISTEN 0 128           [::]:8000       [::]:*   # IPv6 version of 0.0.0.0
```
